A primeira coisa a ser iniciada é o FS Tracker Protocol no servidor.

(REG) Quando um 'node' quer se ligar, inicia o FS_Node, este envia uma mensagem de registo para o servidor.
(Uma ideia a testar é enviar a lista (LIST) de ficheiros e blocos disponíveis juntamente com a msg de REG) 

(ACK) O servidor ao receber esta mensagem de registo, deve devolver um acknowledgment.

O 'node' apenas pode começar a fazer qualquer outra coisa quando receber o ACK do seu registo no servidor.
Tem um tempo de espera até o receber, caso contrário tenta um novo envio de REG.

(AVF-Available Files) A primeira informação a ser trocada após ACK do registo é a lista de ficheiros e blocos disponíveis.
O servidor devolve mais um ACK

(GET) Comando utilizado quando um 'node' quer transferir um ficheiro.
Tem como argumento o nome do ficheiro, quando pedido o servidor deve retornar uma lista de 'nodes' e blocos neles disponíveis.
O escalonamento via carga de node ocorre no servidor, para não mandar sempre o mesmo node como melhor opção de transferencia
Depois o cliente terá um algoritmo de escolha conforme a informação que recebeu mais a sua conexão com os 'nodes'

O Node deverá devolver ao servidor o(s) node(s) que escolhe para este ter sempre conhecimento da carga em cada um.



(List - comando que um 'node' pode enviar ao servidor para saber que ficheiros estão disponíveis para transferencia)
